⚡ Optimize BTC hedge exposure calculation to fix N+1 problem

💡 **What:**
- Refactored `_btc_hedge_exposure` and `_position_payload_price` in `htxbot/combined.py` to prevent synchronous API calls inside loops.
- Extracted a helper `_extract_payload_price` to fetch inline prices safely from position payloads without invoking the exchange.
- Added a pre-scan of missing symbol prices to batch fetch them globally with a single `fetch_tickers` call.
- Maintained fallback individual `fetch_ticker` lookups for complete backwards compatibility and safety.

🎯 **Why:**
- Fixed a textbook N+1 network delay issue where the bot was issuing blocking, synchronous `fetch_ticker` calls repeatedly in a loop for each symbol missing price data in its local payload.
- This dramatically increases execution performance when analyzing portfolios with many open positions lacking inline prices.

📊 **Measured Improvement:**
- Benchmark execution time reduced significantly.
- Baseline implementation with simulated 0.1s API delay for 10 missing prices: ~1.00s.
- Optimized implementation with batched 0.15s API delay for 10 missing prices: ~0.15s.
- Represents an 85% reduction in execution time for this code path.
